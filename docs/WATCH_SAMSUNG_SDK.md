# WATCH_SAMSUNG_SDK.md — 갤럭시워치 러닝 연동 실현가능성 검토

> 회장 결정(2026-07-09): 워치 러닝을 제대로 읽기 위해 **삼성헬스 직접 연동(SDK) 검토**. 이번 완성 사이클 밖, 별도 과제.
> 배경: adb 실증으로 **삼성헬스가 러닝 운동세션·거리를 Health Connect에 안 쓴다**가 확정([`JN.md`](../JN.md) 워치 진단 참조). 우리 앱·코드는 정상.

---

## 1) 핵심 발견 — 이건 "우리 버그"도 "영구 한계"도 아닌 **삼성의 알려진 회귀 버그**

조사 결과, 우리가 본 증상(걸음수·심박수는 HC에 오는데 **운동·거리만 안 옴**)은 **전 세계 갤럭시 사용자 공통 문제**이고 삼성이 공식 인정했다:

- 특정 삼성헬스 업데이트(보고된 버전 `7.00.0.107` 등) 이후 **삼성헬스가 운동(workout)을 Health Connect에 쓰지 않게 됨.** 걸음수·심박수는 정상 동기화. → **우리 관측과 100% 일치.**
- **삼성 공식 응답**: 버그 인정, "향후 앱 업데이트로 수정 예정"(구체 일정 미공지).
- 사용자 워크어라운드: ①삼성헬스 **구버전 롤백** ②서드파티 앱 **"Health Sync"** 경유.

**시사점**: Health Connect는 올바른 표준 아키텍처이고, **우리 앱은 이미 정상 작동**한다. 삼성이 HC 쓰기 버그를 고치면 **코드 한 줄 안 바꿔도 워치 러닝이 자동으로 들어온다.** 지금 급히 SDK로 갈아탈 이유가 약하다.

## 2) 삼성헬스 Data SDK 직접 연동 — 실현가능성

HC를 우회해 삼성헬스에서 운동을 직접 읽는 경로. 조사 결과 **지금은 권장하기 어렵다.**

| 항목 | 내용 | 평가 |
|---|---|---|
| SDK 종류 | 구 `Samsung Health SDK for Android` **2025-07 deprecated** → 신 **`Samsung Health Data SDK`** 로 이전됨 | 신 SDK 대상 |
| 읽기 개발/테스트 | **개발자 모드**로 파트너 승인 없이 운동 읽기 개발·테스트 가능 | ✅ 가능 |
| **배포(실사용)** | 앱 배포하려면 **파트너십 승인** 필요(패키지명 + SHA-256 서명 등록) | ⚠️ 관문 |
| **외부 블로커** | **2025년 기준 삼성이 파트너 프로그램 개편 중 — 신규 파트너 신청 접수 중단.** | 🚫 **현재 승인 자체가 막힘** |
| Expo/RN 적합성 | JS SDK 없음. **네이티브(Kotlin) 모듈 + Expo config 플러그인** 자작 필요. dev/preview build 전용 | ⚠️ Garmin급 공수 |
| 결론 | 개발은 가능하나 **배포 승인이 외부 사정으로 막혀 있어** 지금 완주 불가 + 큰 네이티브 공수 | **보류 권장** |

## 3) 선택지 비교

| 경로 | 공수 | 리스크/블로커 | 지금 할만한가 |
|---|---|---|---|
| **A. HC 유지 + 삼성 수정 대기** | 0 (코드 이미 완성) | 삼성 수정 일정 불확실 | ✅ **기본값** — 아키텍처 올바름, 자동 복구 |
| **B. 삼성헬스 Data SDK 직접** | 큼(네이티브 모듈) | **파트너 승인 접수중단(외부)** + deprecated 이력 | ❌ 지금은 막힘 |
| **C. 워크어라운드(롤백/Health Sync)** | 소(사용자측) | 배포 불가·불안정·사용자 부담 | ❌ 제품화 부적합 |
| **D. Wear OS 컴패니언 앱** | 큼 | 갤럭시워치=Wear OS, 워치앱이 HC에 직접 쓰기 가능하나 별도 앱 개발 | 🔮 장기 옵션 |

## 4) 권고

1. **워치 러닝 연동은 "HC 경로 유지 + 삼성 수정 대기"를 기본**으로 둔다. 우리 앱은 이미 올바르게 구현돼 있어, 삼성이 HC 쓰기 버그를 고치는 순간 **무빌드로 동작**한다. (실제로 삼성헬스 업데이트 후 재테스트만 하면 됨.)
2. **삼성헬스 Data SDK 직접 연동(B)은 "파트너 프로그램 재개 시 착수"하는 후행 과제**로 등록. Garmin과 같은 "벤더 직접연동" 묶음. 지금은 외부 승인 블로커로 완주 불가.
3. **당장의 러닝 기록 주 경로 = 앱 내 GPS 트래킹**([러닝 시작], 지도·거리·페이스). 이건 벤더 의존 없이 100% 동작. 완성 사이클은 GPS로 닫을 수 있다.
4. **사용자 커뮤니케이션**: 워치 화면에 "삼성헬스 운동→Health Connect 동기화는 삼성 측 이슈로 일시 제한. 삼성헬스 업데이트 후 자동 연동됩니다. 지금은 [러닝 시작] GPS를 이용하세요" 같은 안내 문구를 두면 사용자 혼란 방지(후속 UX 개선 시).

## 5) 재검증 트리거 (나중에 이걸로 확인)

- 삼성헬스 앱 업데이트 후: 러닝 1회 → Health Connect **데이터 및 액세스 → 활동에 "운동"·"거리"가 생기는지** 확인(`adb shell am start -a android.health.connect.action.MANAGE_HEALTH_DATA`). 생기면 [워치 불러오기] 즉시 성공.
- 삼성 파트너 프로그램 재개 공지 확인 시 → B(Data SDK) 착수 재검토.
