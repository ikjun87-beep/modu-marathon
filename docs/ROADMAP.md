### 🔴 P0 — 확산이라는 목표 자체가 지금 성립 안 한다 (제품·웹 렌즈 수렴)

**P0-1. 웹에 앱 설치 경로가 전무하다.**
히어로·나브·모바일 스티키바의 모든 CTA가 "앱 출시 알림 받기"(이메일 waitlist)로 수렴하고, APK 다운로드 링크는 0개. 앱이 이미 존재하고 워치 연동까지 완결됐는데 공개 웹은 "앱을 준비 중이에요"(사실과 다른 카피)라고 말한다. 게다가 현행 배포 방식(카톡으로 APK 전달)은 **모바일 카톡이 .apk를 차단**해서 작동하지 않고, 갤럭시 One UI 6+의 **Auto Blocker 기본 ON**이 사이드로드를 한 번 더 막는다. → **`/install` 페이지 신설**(APK 다운로드 + "출처를 알 수 없는 앱"·Auto Blocker 해제 안내 + 아이폰은 "웹으로 이용" 안내) + 히어로 CTA를 "앱 받기"로 교체. APK 호스팅은 GitHub Releases(무료·용량 무제한) 우선 검토 — Firebase Hosting 무료플랜은 일 전송 360MB라 113MB APK면 하루 3건에 소진.

**P0-2. 사이드로드인데 업데이트 배포 수단이 없다.**
버그픽스 한 번마다 크루원 전원에게 "지우고 다시 깔아라"를 시켜야 한다("또 깔라고?" 두 번이면 이탈). → Firestore `meta/latest` 문서(versionCode·url·note) + 앱 홈 상단 "새 버전" 배너. 반나절 작업, 사이드로드 체제의 생명유지장치.

**P0-3. 일정이 앱 바이너리에 하드코딩 — 8/2 이후 시한폭탄.**
`events.ts`의 마지막 모임이 8/2이고 `nextEvent()`는 다가올 모임이 없으면 지난 모임을 계속 보여준다 → 2주 뒤부터 홈 1번 카드가 영구히 낡은 모임 표시("이 크루 망했나?" 신호). 일정 추가 = 코드 수정+재빌드+전원 재설치(P0-2와 곱해짐). 웹도 같은 하드코딩이라 **7/5 지난 모임에 아직도 참석 신청이 가능**(유령 참석 데이터 생성). → `events` Firestore 이관(기존 id 유지 = 참석 기록 무손실), 웹·앱 공용. **"일정 만들기"가 로드맵 실질 1번인 진짜 이유는 편의가 아니라 이것.**

### 🟠 P1 — 신뢰·확산 보조 (지금 고침, 전부 작음)
- **죽은 오픈채팅 버튼 3개**(히어로·합류 섹션, "곧 열려요" 토스트 몇 주째) — 실제 오픈채팅방 링크 연결 또는 버튼 제거. 첫 화면의 죽은 버튼은 사이트 전체를 미완성으로 보이게 함.
