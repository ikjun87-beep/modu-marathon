# UX_APP_NAV.md — 앱 네비게이션·다중 페이지 개선 제안

> 회장 관심사(2026-07-09): *"스마트스코어 앱은 되게 부드러운데 앱에 여러 페이지가 있는거지? 참고할 수 있는 앱이 있으면."*
> → 현재 **2탭(크루·러닝)** 구조를 스마트스코어·Runna식 **다중 페이지 + 부드러운 전환**으로 키우는 제안.
> **실행 시점 = 완성 사이클(feat/m3 → main 머지) 종료 후 "P7 UX 개선 사이클"**. 이 문서는 그 승인용 청사진.

---

## 1) "부드럽다"의 정체 — 감각이 아니라 구체 기술 요소

회장이 스마트스코어에서 느낀 "부드러움"은 취향이 아니라 아래 요소들의 합이다. 우리가 그대로 이식할 수 있다.

| 느껴지는 것 | 실제 기술 요소 | 우리 앱 현재 | 이식 방법(Expo) |
|---|---|---|---|
| 화면이 밀려 들어옴 | **push 스택 전환**(리스트→상세) | 모달만 있음, 페이지 없음 | Expo Router `Stack` (탭마다 하위 스택) |
| 탭 전환이 안 끊김 | 네이티브 탭 + 상태 보존 | `NativeTabs` 씀(양호) | 유지 + 하위 스택 결합 |
| 로딩이 안 답답함 | **스켈레톤 UI**(빈 화면 대신 회색 뼈대) | 빈 화면/스피너 | 간단 Skeleton 컴포넌트 |
| 눌렀을 때 반응 | **터치 피드백**(scale·opacity) | 기본 Pressable | `Pressable` press 스케일 애니 |
| 데이터가 살아있음 | **홈 큐레이션 카드**(개인화 요약) | 러닝 탭에 숫자만 | 홈 탭 신설(아래) |
| 뭐든 빨리 찾음 | **통합 검색** | 없음 | 검색 바(크루·러닝·모임 통합) |
| 스크롤이 쫀득 | 리스트 가상화·관성 | FlatList 씀(양호) | 유지 |

**핵심**: 스마트스코어의 부드러움 = ①push 상세 페이지 ②스켈레톤 ③터치 애니 ④홈 큐레이션. 네 가지가 8할이다.

## 2) 레퍼런스 앱 — 무엇을 볼지 콕 집어서

| 앱 | 우리가 훔칠 것 | 보는 법 |
|---|---|---|
| **스마트스코어**(골프) | 홈 개인화 큐레이션, 하단 중앙 강조 버튼(MY스코어), 성과 배지=정체성, 통합 검색 | 홈 화면 스크롤·MY스코어 탭 |
| **Runna**(러닝) | 5탭 IA(Today·Plan·Activities·…), "오늘 할 것" 중심 홈 | Today 탭 구성 |
| **Nike Run Club** | 마찰 최소화, 러닝 상세(지도·스플릿·페이스 그래프) 화면 | 러닝 종료 후 상세 |
| **Strava** | 활동 피드(크루가 서로의 러닝을 봄), 세그먼트·기록 상세 | 홈 피드·활동 상세 |
| **토스/캐치테이블**(국내 부드러움 표준) | 전환 애니·스켈레톤·마이크로 인터랙션의 한국형 결 | 화면 전환 관찰 |

> 회장이 직접 볼 때 관찰 포인트: **"리스트 하나 탭했을 때 어떻게 다음 화면으로 넘어가나"** — 이 push 전환이 부드러움의 뼈대다.

## 3) 제안 IA — 2탭 → 5탭 + 하위 페이지

```
현재                      제안 (스마트스코어/Runna 식)
┌─────────┐              ┌──────────────────────────────────────────┐
│ 크루     │              │ 홈  │ 크루 │ 러닝 │ 랭킹 │ 마이            │  ← 하단 5탭
│ 러닝     │      →       └──┬──┴──┬──┴──┬──┴──┬──┴──┬─────────────┘
└─────────┘                 │     │     │     │     └ 프로필/통계/배지/설정/처리방침
                            │     │     │     └ 크루 주간 거리 랭킹·챌린지(배지)
                            │     │     └ 러닝 기록 → [상세] 지도·스플릿·심박 (push)
                            │     └ 방명록·갤러리·모임참석 → [모임 상세] (push)
                            └ 오늘 요약 큐레이션 + 다가오는 모임 + 빠른 러닝 시작
```

### 각 탭 구성

**① 홈 (신설 · Today)** — 스마트스코어 홈 큐레이션 이식
```
┌────────────────────────────┐
│ [통합 검색 ▸ 크루·러닝·모임]  │
│ ─────────────────────────  │
│ 오늘 뛴 거리  3.2km  🔥      │  ← 개인화 요약 카드
│ 이번 주 크루 합계  128km     │
│ [▶ 러닝 시작] [⌚ 워치]      │  ← 빠른 실행(하단 중앙 강조=MY스코어 패턴)
│ ─────────────────────────  │
│ 📅 다가오는 모임  토 07:00   │  → 탭하면 모임 상세(push)
│ 💬 크루 새 글 3              │  → 크루 탭으로
└────────────────────────────┘
```

**② 크루** — 기존 `index.tsx` 유지 + 모임 상세 페이지(push) 추가
**③ 러닝** — 기존 `explore.tsx` 유지 + **러닝 상세 페이지 신설**(현재 히스토리는 요약만 → 탭하면 지도·스플릿·페이스·심박 그래프)
**④ 랭킹/챌린지 (신설)** — 크루 주간 거리 랭킹, 이달의 챌린지, **성과 배지**(첫 5km·주 3회·월 100km) = 스마트스코어 "골퍼 정체성" 패턴의 러너 버전
**⑤ 마이 (신설)** — 내 총 통계, 배지 진열, 온보딩 이름 수정, 처리방침 링크(→ modu-marathon.web.app/privacy), 설정

## 4) 단계별 실행안 (P7 UX 개선 사이클 · 머지 후)

작은 단위로, 각 단계가 독립 배포 가능하게.

| 단계 | 내용 | 크기 | 선행 |
|---|---|---|---|
| **7-1** | **러닝 상세 페이지**(push) — 기록 탭하면 지도·스플릿·페이스. 부드러움 첫 체감 | S | 없음(가장 임팩트 큼, 여기부터) |
| **7-2** | **터치 애니 + 스켈레톤** 공통 컴포넌트 — 전역 부드러움 | S | 없음 |
| **7-3** | **홈 탭 신설**(큐레이션) — 5탭 중 홈 먼저 | M | 7-1 |
| **7-4** | **마이 탭**(프로필·배지·설정·처리방침 링크) — 후행 TODO였던 온보딩 처리방침 링크도 여기서 흡수 | M | 없음 |
| **7-5** | **랭킹/챌린지 + 배지** — 크루 리텐션(스마트스코어 정체성). Firebase 집계 | L | Firebase(완료됨) |
| **7-6** | **통합 검색** — 마지막 완성도 | M | 7-3 |

> **권장 시작점 = 7-1 러닝 상세 페이지.** 데이터·지도 컴포넌트(`run-map`)가 이미 있어 재활용 가능하고, "리스트→상세 push"라는 부드러움의 핵심을 가장 싸게 체감시킨다.

## 5) 리스크·주의

- **NativeTabs + 하위 스택 결합**: 현재 `expo-router/unstable-native-tabs` 사용 중. 탭 안에 `Stack`을 두는 구조가 Expo Router에서 검증돼 있으나, `unstable` API라 SDK 업데이트 시 확인 필요. → 7-1 착수 전 Expo v57 문서 재확인(`app/AGENTS.md` 규칙).
- **스코프 크립 경계**: 이건 **완성 사이클(설치·확산) 다음**의 리텐션 개선이다. 머지·배포가 먼저. 배지·랭킹은 사용자 실제 유입 후 가치가 커지므로 7-5는 유저 확보 뒤로 미뤄도 됨.
- **좌우 슬라이드 네비**(스마트스코어)는 우리 규모엔 과함 → 하단 5탭이면 충분. 슬라이드는 채택 안 함.

## 6) 한 줄 결론

> 부드러움 = **push 상세 페이지 + 스켈레톤 + 터치 애니 + 홈 큐레이션** 4종. **머지 후 7-1(러닝 상세)부터** 작은 단위로. 레퍼런스는 러닝 상세=NRC, 홈 큐레이션=스마트스코어, 전환 결=토스.
