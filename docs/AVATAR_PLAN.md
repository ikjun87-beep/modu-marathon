# AVATAR_PLAN.md — 마스코트 꾸미기 기획서

> 회장 요청(2026-07-17): "캐릭터를 남/여로 고르고 색상도 고르게, 옷(상의·하의)·신발도 꾸미게 — 예전 아바타 꾸미듯이."
> **결정: 지금 착수 아님.** 로드맵(설치 퍼널·확산)이 먼저. 이 문서는 그 뒤 바로 시작할 수 있게 방법·순서·규모를 확정해 둔 것.

---

## 1) 왜 지금 안 하나 (정직하게)

- **리텐션 기능이지 획득 기능이 아니다.** "있으면 재밌는 것"이지 "없으면 안 쓰는 것"이 아니다. 지금 최우선은 "크루 10명이 앱을 **깔** 수 있게"인데(웹에 설치 경로가 없음 — ROADMAP §3 P0), 깔지도 못하는 상태에서 옷장을 만들면 **아무도 못 보는 옷장**이 된다.
- **선행 조건이 크다.** 지금 마스코트는 AI가 그린 납작한 PNG라 부위별 색을 못 바꾼다(아래 §2). 벡터 재설계가 전제.
- **제값이 나오는 시점**: 크루가 실제로 매일 앱을 여는 단계 → "내 캐릭터를 남과 다르게" 욕구가 생김 → 그때 꾸미기가 리텐션으로 작동. 그 전엔 투자 대비 0.

## 2) 지금 구조로 왜 안 되나 — 기술 사실

- 현재 마스코트 = `assets/images/mascot-{male,female}.png` (512px 래스터). 남녀 2종은 **하나의 원본에서 코드로 색만 바꿔** 파생(`scripts/build-mascot-pair.mjs`) — 이건 빌드 타임 1회 계산이지 런타임 조합이 아니다.
- 래스터 PNG는 "여기가 상의, 여기가 신발"이라는 **부위 정보가 없다.** 런타임에 상의만 빨강으로 칠하려면 그 픽셀이 상의인지 앱이 알아야 하는데 PNG엔 그 정보가 없다.
- RN에서 런타임 부분 리컬러의 현실적 유일 해법 = **부위별로 분리된 SVG 레이어**(`react-native-svg`, 이미 의존성 있음).

## 3) 해야 하는 일 (순서·규모)

### P0. 마스코트를 부위별 벡터로 재설계 — **이 프로젝트의 8할**
현재 PNG(V1 확정본, 엄지척 러너)를 기준으로, 색이 바뀌는 부위를 **따로 채색되는 SVG `<Path>`**로 분리:
- `skin`(머리·팔·다리) · `top`(민소매 상의) · `bottom`(반바지) · `shoes`(운동화) · `outline`(검은 선, 불변) · `face`(눈·입·볼터치, 불변)
- 방법 후보:
  - (a) **손으로 벡터화** — V1 PNG를 일러스트레이터/Figma에서 부위별 path로 다시 그림. 품질 최고, 시간 최대.
  - (b) **AI 벡터 생성 후 부위 분리** — gpt-image-1은 SVG를 못 뱉으니, PNG를 potrace 등으로 벡터화 → 부위별 색역으로 path 분리(build-mascot-pair.mjs의 색 분류 로직 재활용 가능). 반자동.
  - (c) **레이어 PNG 세트** — 부위별 흑백 마스크 PNG를 만들고 `tintColor`로 런타임 채색해 겹침. SVG보다 거칠지만 RN Image tint로 가능. 절충안.
- **권고: (c) 절충안으로 시작** — build-mascot 파이프라인이 이미 부위 색을 분류하니, 그걸 **마스크 PNG로 추출**하는 게 제일 가깝다. 품질 부족하면 (a)로.

### P1. 옷장 데이터 모델
```
avatar(기기 로컬 + 계정 동기화)  { base:'male'|'female', skin, top, bottom, shoes }  // 각 색 hex 또는 프리셋 id
```
- 지금 `lib/mascot.ts`(male/female 선택)를 확장. **취향이라 서버 필수는 아니나**, 크루원에게 보이려면(참석 목록·랭킹의 내 캐릭터) 문서에 얹어야 한다 → uid 소유 기반으로 간 뒤가 깔끔(ROADMAP §4와 맞물림).
- 색은 **자유 hex 아님, 프리셋 팔레트**(6~8색). 자유 색은 브랜드 톤을 깨고 조합 지옥.

### P2. 꾸미기 UI
- 마이 탭 "내 캐릭터"를 확장: 미리보기(실시간 반영) + 부위 탭(상의/하의/신발) + 색 팔레트.
- `useMascot` 구독 패턴 그대로 — 고르면 앱 전역 즉시 반영(이미 됨).

### P3. 노출
- 크루 참석 목록·랭킹·댓글에 이름 대신(또는 옆에) 내 캐릭터 썸네일. **이게 있어야 꾸밀 동기가 생긴다** — 아무도 안 보면 안 꾸민다.

## 4) 규모 감각
- P0(벡터/마스크 재설계)만으로 지금까지 마스코트 작업 전체의 2~3배. P1~P3 합치면 **배지+마스코트+폰트를 합친 것과 비슷한 사이클**.
- 그래서 **크루가 실제로 쓰기 시작한 뒤** 착수가 맞다. 지금 하면 순서가 뒤집힌다.

## 5) 지금 당장 한 것 (이 요청에서 분리 처리)
- **하단 탭 아이콘 색** = 선택 탭 브랜드 블루(`app-tabs.tsx` tintColor·iconColor). 꾸미기와 무관하게 즉시 반영. (당근 하단바 톤)
- 남녀 선택은 **이미 됨**(마이 탭). 색·옷·신발이 이 문서의 범위.
